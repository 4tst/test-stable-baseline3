# 贪吃蛇 AI - 实现总结

## ✅ 已完成的功能

### 1. AI 训练系统
- **算法**: PPO (Proximal Policy Optimization)
- **框架**: Stable-Baselines3
- **策略网络**: MLP (多层感知机)
- **训练步数**: 500,000 步
- **模型保存**: `ppo_snake_v1.zip`

### 2. AI 测试系统
- **自动加载**: 从保存的模型文件加载
- **实时渲染**: Pygame 图形界面展示
- **多局统计**: 记录每局得分和最高分
- **帧率控制**: 15 FPS，流畅观看
- **退出机制**: ESC 键或关闭窗口

### 3. 优化的超参数
```python
learning_rate=3e-4      # 学习率
n_steps=2048            # 采样步数
batch_size=64           # 批量大小
n_epochs=10             # 训练轮数
gamma=0.99              # 折扣因子
gae_lambda=0.95         # GAE 参数
clip_range=0.2          # PPO 裁剪范围
ent_coef=0.01           # 探索系数
vf_coef=0.5             # 价值函数系数
max_grad_norm=0.5       # 梯度裁剪
```

### 4. 观察空间设计 (8维)
1. 蛇头 X 坐标（归一化）
2. 蛇头 Y 坐标（归一化）
3. 食物 X 坐标（归一化）
4. 食物 Y 坐标（归一化）
5. 前方危险检测
6. 预留维度
7. 预留维度
8. 后方危险检测

### 5. 动作空间设计 (3种)
- **0**: 直走（保持当前方向）
- **1**: 右转（顺时针 90°）
- **2**: 左转（逆时针 90°）

### 6. 奖励函数设计
| 事件 | 奖励值 | 目的 |
|------|--------|------|
| 吃到食物 | +10 | 鼓励寻找食物 |
| 撞墙/撞自己 | -10 | 避免死亡 |
| 每步移动 | -0.1 | 鼓励快速行动 |
| 超时死亡 | -5 | 防止苟活策略 |

## 💡 AI 工作原理

### 决策过程
```
当前状态 (8维向量)
    ↓
输入神经网络
    ↓
输出动作概率分布
    ↓
选择最优动作
    ↓
执行动作
    ↓
获得新状态和奖励
    ↓
更新策略（训练时）
```

### 学习机制
1. **探索**: 通过熵系数鼓励尝试新动作
2. **利用**: 选择高奖励的动作序列
3. **泛化**: 从相似状态中学习通用策略
4. **优化**: PPO 算法保证稳定更新

## 🔧 性能优化建议

### 加速训练
1. **使用 GPU**
   ```python
   model = PPO("MlpPolicy", env, device="cuda")
   ```

2. **并行环境**
   ```python
   from stable_baselines3.common.env_util import make_vec_env
   env = make_vec_env(lambda: SnakeEnv(), n_envs=4)
   ```

3. **减少日志**
   ```python
   model = PPO(..., verbose=0)  # 关闭详细输出
   ```

### 提高性能
1. **增加训练步数**
   ```python
   model.learn(total_timesteps=1000000)
   ```

2. **调整超参数**
   - 降低学习率 → 更稳定
   - 增加批量 → 更准确
   - 增加探索 → 更多样

3. **改进观察空间**
   - 添加距离信息
   - 添加方向编码
   - 使用图像输入 (CNN)

## ❓ 常见问题

### Q: AI 不学习怎么办？
**A**: 
- 检查奖励函数是否正确
- 增加训练步数
- 调整学习率
- 确保环境没有 bug

### Q: 如何提高 AI 水平？
**A**:
- 训练更多步数
- 使用更好的超参数
- 改进观察空间
- 多次训练取最优

### Q: 训练太慢怎么办？
**A**:
- 使用 GPU
- 减少训练步数先测试
- 并行多个环境
- 降低 verbose 级别

### Q: AI 表现不稳定？
**A**:
- 继续训练更多步数
- 调整探索系数
- 使用不同的随机种子
- 检查是否有过拟合
